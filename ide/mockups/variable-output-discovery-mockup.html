How it works:

  1. User clicks an input field → dropdown shows only compatible variables
  2. User clicks "(show all ▼)" → expands to show ALL variables
  3. Incompatible variables appear grayed out with ⚠ warning icon
  4. Hovering shows tooltip: "Type mismatch: expected auth_token, got string"
  5. User can still select them (advanced override) — we warn but don't block
✅ Beginners are protected by smart defaults. Advanced users retain full control.

Input definition (block JSON)

{
  "name": "edr_token",
  "accepts": ["auth_token"],
  "type": "string"
}
{
  "name": "dataplane_url",
  "accepts": ["dataplane_url", "url"],
  "type": "string"
}
{
  "name": "method",
  "type": "string"
  // no "accepts" → shows all variables
}
💡 Additive change — blocks without class / accepts keep working as before. The string class is the implicit default for untyped outputs.
